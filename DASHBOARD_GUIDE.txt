FORM FIELDS EXPLAINED:

CREATE CASE:
  Case Name *: What to call this investigation
  Case Type *: hawala, structuring, money_laundering, etc.
  Priority *: high, medium, low
  Currency: PKR, USD, EUR, GBP (default: PKR)
  Description: Additional notes about the case

ADD TRANSACTION:
  Select Case *: Which case this transaction belongs to
  Amount *: Money involved (numbers only)
  Currency: PKR, USD, EUR (default: PKR)
  Source Entity *: Where money comes from (e.g., bank/person ID)
  Destination Entity *: Where money goes to (e.g., bank/person ID)
  Transaction Date: When it happened

ADD THREAT:
  Threat ID *: Unique identifier (e.g., THREAT_001)
  Threat Type *: sanctions, pep, criminal, fraud, terrorist_financing
  Risk Level *: critical, high, medium, low
  Entity Name *: Person or organization name
  Details: Additional threat information

TROUBLESHOOTING:

Q: Forms not showing?
A: Click "‚ûï Add New Data" button at top

Q: "Error connecting to server"?
A: Make sure python aml_system_v6_enhanced.py is running
   Check that port 5000 is not blocked

Q: Form submission failed?
A: Check that all * (required) fields are filled
   Try using different data
   Check browser console (F12) for errors

Q: New data not appearing?
A: Click "üîÑ Refresh Data" button
   Wait a moment for database to update
   Clear browser cache if needed
